# severlet

# 编写severlet的原理：

  1. 首先是通过sun公司的severlet接口，通过规定的一套接口与tomcat服务器进行连接

  2. 其次是需要tomcat的服务器进行交互，实现severlet接口与服务器的连接，以及tomcat服务器与我们程序员自己编写的webapp进行交互

  3. 程序员所需要做的事就是编写webapps，主要是两种东西

    • 实现了severlet接口的java类
    • 编写配置文件，通过hashmap的方式进行存储对应的键值对，由配置文件进行解耦合，实现多个键值对的形成（其中就包含了反编译，对配置文件的反编译获得newinstance，然后将此对象进行解耦合）

# 出现的问题：不用maven创建tomcat服务

  • 解决步骤：
  • 直接在idea中创建工程，普通的java工程；
  • 点击项目中的add framework support 然后添加webapplication的服务框架
  • 在右上角的地方，edit configurations 然后添加tomcat服务器->tomcat ->local
  • 重点：在file - project structure 中有一个dependence的选项，这里面是添加jar包依赖的地方，左上角的小加号中直接选取

# 服务器端的操作：

1.Web Client向Servlet容器(Tomcat)发出Http请求 2.Servlet容器接收Web Client的请求 3.Servlet容器创建一个HttpServletRequest对象，将Web Client请求的信息封装到这个对象中 4.Servlet容器创建一个HttpServletResponse对象 S.Servlet容器调HttpServlet对象service方法，把Request与Response作为参数，传给HttpServlet 6.HttpServlet调用HttpServletRequest对象的有关方法，获取Http请求信息7.HttpServlet调用HttpServletResponse时象的有关方法，生成响应数据 8.Servlet容器把HttpServlet的响应结果传给Web Client

# request常用方法：

最重要的方法：getParameter("对应的参数")；只要用serverlet传递请求，就可以用这个方法来获取信息。

  • 请求转发：（是req的一种方法）

    • 解释：是一种服务器的行为，当客户端请求到达后，服务器进行转发，此时会将请求对象进行保存，地址栏中的URL地址不会改变，得到响应后，服务器端再将响应发送给客户端，从始至终只有一个请求发出。

    • 特点：共享数据，同一个url；一个请求；请求转发时的地址只能是当前站点下的（当前项目）的资源；

    • 方式： RequestDispatcher getRequestDispatcher(String path)

  • 乱码问题只要在前面加上：

    • 请求乱码：request.setCharacterEncoding("utf-8");
    • new String(str.getBytes("iso8859-1"),"utf-8"); //首先把得到的数据转换成iso8859-1的二进制 再把二进制进行utf-8编码 对于get请求只能手动编解码解决乱码问题。

# servlet 的response 对象：

  1. 可以响应数据：两种流

    1. getWriter() 字节流 只能输出字节
    2. getOutputStream() 字符流 可以输出一切数据
    3. 两者不能够同时使用；否则会报错：response对象已经响应了请求，不会响应另一个

  2. 响应乱码：

    解决方案：

    • 既要设置服务端的编码，也要设置客户端的编码：
    • resp.setCharacterEncoding("utf-8"); 设置服务端的编码
    • resp.setHeader("content-type","text/htlm;charset = utf-8"); 客户端的编码；首先读取格式为html页面；然后响应编码
    • 最终解决方案：一句同时设置服务端和客户端编码：resp.setContentType("text/html;charset = utf-8");

  3. 重定向（是resp的一种方法）:

    • 解释：是服务端的操作，客户端的 行为：经典是就是 还有几面跳转到下一个页面

    • 特点：服务器接受响应的同时，会发出另一个请求；但是这个和请求的区别是他并不会共享数据，而且总共是造成了两次的数据请求；URL同时也会变化；地址可以是任何资源

    • 方法：resp.sentRedirect(url)；

    • 状态：第一个的响应码是302；跳转的目标页面是200

# cookie

# 1.cookies的创建以及获取：

  1. 还是通过Servlet服务来创建；

  2. Cookie cookies = new Cookie("name","value");
//这里因为是创建cookie；所以还是响应对象；用resp
resp.addCookie(cookies)

  3. 接受cookies：

    1. 接受的cookies是一个String类型的数组；

    2. //因为这里是获取请求；获取cookie的值：用req对象
Cookie []cookies =req.getCookies();
if (cookie!=null&&cookie.length>0)
     for (Cookie cookies :cookie) {
     String name = cookies.getName();
     String value = cookies.getValue();
}

  4. cookies的到期时间：

    1. ·负整数 若为负数，表示不存储该cookie。 cookie的maxAge属性的默认值就是-1，表示只在浏览器内存中存活，一旦关闭浏览器窗口，那么cookie就 会消失。

    2. ·正整数 若大于0的整数，表示存储的秒数。 表示cookie对象可存活指定的秒数。当生命大于0时，浏览器会把Cookie保存到硬盘上，就算关闭浏览 器，就算重启客户端电脑，cookie也会存活相应的时间。

    3. ·零 若为0，表示删除该cookie。 cookie生命等于0是一个特殊的值，它表示cookie被作废！也就是说，如果原来浏览器已经保存了这个 Cookie,那么可以通过Cookie的setMaxAge(O)来删除这个Cookie。无论是在浏览器内存中，还是在客户端 硬盘上都会删除这个Cookie。

  5. 对于同名的Cookie对象：会覆盖原先的

  6. Cookie的路径 1.当前服务器下任何项目的任意资源都可获取Cookie对象 设置路径为”/"，表示在当前服务器下任何项目都可访间到Cooki对象 2.当前项目下的资源可获取Cookie对象（默认不设置Cookie的path） 默认不设置Cookie的path或者设置为当前站点名 3.指定顶目下的资源可获取Cookie对象 4.,指定目录下的资源可获取Cookie对象 总结：只有访间的路径中包含Cookie对象的path值，才可以获取到该cookie对象

# ServletContext

  1. 解释：每一个web应用都有且仅有一个ServletContext对象，又称Application对象，从名称中可知，该对象是与应用程序相关的。在WEB容器启动的时候，会为每一个WEB应用程序创个对应的ServletContext对象。

  2. 两大作用

    1. 第一、作为域对象用来共享数据，此时数据在整个应用程序中共享；

    2. 第二、该对象中保存了当前应用程序相关信息。例如可以通过getServerlnfo()方法获取当前服务器信息，getRealPath(Stringpath)获取资源的真实路径等。

  3. 获取servletcontext的方法：

      ServletContext servletContext = req.getServletContext();//通过req获取
  ServletContext servletContext1 = req.getSession().getServletContext();//通过session获取
  ServletContext servletContext2 = getServletConfig().getServletContext();//通过servletconfig对象获取
  ServletContext servletContext3 = getServletContext();//直接获取

  4. 常用的方法：

    • 获取当前服务器的版本信息：

      String serverInfo = req.getServletContext().getServerInfo();

    • 获取项目的真实路径

      String realPath = request.getServletContext().getRealPath("/");;

  5. 是最大的域对象

    • 步骤：

    • 获取servletcontext

    • 设置域对象setAttribute

    • 获取域对象

    • 移除对象

    • 当然不建议存放过多数据，因为ServletContext中的数据一旦存储进去没有手动移除将会一直保存。

# Servlet三大域对象：->一般就是数据传输

  1. request域对象

    在一次请求中有效。请求转发有效，重定向失效。

  2. sessioni域对象

    在一次会话中有效。请求转发和重定向都有效，session销毁后失效。

  3. .servletContext域对象

    在整个应用程序中有效。服务器关闭后失效
