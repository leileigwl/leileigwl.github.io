# 方法重载

  • 定义：在一个类中，有相同的函数名，但是形参不同的函数
  • 方法的重载的规则：
    • 方法名称必须相同。
    • 参数列表必须不同（个数不同、或类型不同、参数排列顺序不同等）。
    • 方法的返回类型可以相同也可以不相同。
    • 仅仅返回类型不同不足以成为方法的重载。

# Super

  • super关键字可以调用父类的非私有属性/方法

  • //super调用属性
public class Person {//Person类
    protected String name = "xixi";//保护属性name
}

public class Teacher extends Person{//继承Person类
    private  String name = "haha";
    public  void test(){
        System.out.println(this.name);//haha
        System.out.println(super.name);//xixi,通过super关键词调用
    }
}


  • //super调用方法
public class Person {
    void print(){
        System.out.println("Person");
    }
}
public class Teacher extends Person {
    void print() {
        System.out.println("Teacher");
    }
    public void test() {
        //正常的没有写出来的时候就是调用this，而且this关键字很多时候可以省略
        print();//Teacher
        this.print();//Teacher
        super.print();//Person
    }
}

  • super调用构造器的时候必须放在第一行

  • 总结

    • super:注意点： 1.super调用父类的构造方法，必须在构造方法的第一个 2.super必须只能出现在子类的方法或者构造方法中！ 3.super和this不能同时调用构造方法！

    • Vs this:

      • 代表的对象不同： this:本身调用者这个对象

        super:代表父类对象的应用

    • 前提 this:没哟继承也可以使用 super:只能在继承条件才可以使用

    • 构造方法 this();本类的构造 super():父类的构造！

# 重写

  • 重写是子类对父类的允许访问的方法的实现过程进行重新编写, 返回值和形参都不能改变。

  • 相当于是对原来的方法进行拓展/减少

  • 修饰符：范围可扩大，但是不能缩小

  • //静态重写一次print方法
public class A {
   static void print() {
        System.out.println("A->print");
    }
}
class B extends  A{
    static void print() {
        System.out.println("B->print");
    }
}
public static void main(String[] args) {
        A a= new A();
        a.print();//A->print
        A b = new B();//父类引用指向了子类，实际的diao'yong
        b.print();//B->print
}
//静态重写这个发方法只与左边有关，定义的类的类型有关

//非静态重写print方法
class B {
    void print() {
        System.out.println("B->print");
    }
}
class A extends B {
    void print() {
        System.out.println("A->print");
    }
}
 public static void main(String[] args) {
        A a = new A();
        a.print();//A->print
        B b = new B();
        b.print();//B->print
        B c = new A();//新建一个B的对象，他实际是指向子类A
        c.print();//A->print
}
//要点：子类重写父类，就调用子类的，子类没有重写，那么就调用父类的
  • 注意事项：
    • static 和final，private关键字修饰的方法是无法重写的

# 引用类型转换

  • 声明的类型如果范围较大，可以自动转换，如果是小类型到大类型，就需要强制类型转换，

  • class  Person{
}
class Stu extends Person {
    void show(){
        System.out.println("show");
    }
}
public static void main(String[] args) {
        Person pre = new Stu();
    	pre.show();//会报错，因为Person类里面是没有这个方法的，而是在他的子类里才有
        ((Stu) pre).show();
}

  • 大意：

    • 父类引用指向子类的对象
    • 把子类转换为父类，向上转型
    • 把父类转换成子类，向下转行，强制转换
    • 方便方法的调用，减少重复的代码->低耦合

# 成员内部类

  • 定义在一个类里面的类

  • 特点：可以调用类的私有属性

  • class Outer {
    private int id = 10;
    public void out() {
        System.out.println("这是外部类的方法");
    }
    public class Inner {//内部类的声明
        public void in() {
            System.out.println("这是内部类的方法");
        }
        public int getId() {//获取私有属性
            return id;
        }
    }
}
public static void main(String[] args) {
        Outer outer = new Outer();//先创建外部类的对象
        Outer.Inner inner = outer.new Inner();//通过外部类的new 来创建成语内部类的对象
        inner.in();//"这是内部类的方法"
        System.out.println(  inner.getId());//10
}

# 异常机制

  • 指在程序中一些不期而至的异常：文件找不到，网络链接异常，非法参数；

  • 异常处理框架

  • 解决：

    • 抛出异常

    • 捕获异常

    • 异常处理五个关键字：

      • try,catch,finally(无论遇不遇都会执行，可以不写，一般用来处理善后工作，如关闭io流资源),throw,throws
    • //捕获异常 
public static void main(String[] args) {
        int a = 1,b =0;
        try {
            System.out.println(a/b);
        }catch (ArithmeticException e){//catch里面的是捕获异常的类型，可以多个捕捉异常
            System.out.println(e);//把ArithmeticException异常视作e，输出e
        }finally {
            System.out.println("finally");//每次最后都会有finally执行
        }
}
//要领，如果能遇到异常，不会一下子停止，而是会继续进行下去

    • 快捷：ctrl+alt+t,将选中的代码块进行捕获选择

    • //throw抛出异常->一般用在方法体中
public class Demo04 {

    }
    public  void test(int x ,int y){
        if(y==0){
            throw new ArithmeticException();//直接抛出红色的异常不进行处理，throw是在方法里的
        }
        System.out.println("x/y="+x/y);//如果有异常，不会输出这句
    }
}
